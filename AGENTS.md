# agent.md（项目开发约定）

本文档用于指导 Code Agent 在本仓库内进行后续开发，确保实现方向与现有项目目标、架构和部署方式一致。

## 1) 项目目的

- 本项目是给**我家孩子及其班级同学**使用的学习小工具集合。
- 典型方向包括：学科类学习工具、课堂互动工具、练习/小游戏、辅助教学页面等。
- 设计优先级：
  1. 孩子容易上手
  2. 老师/家长容易在课堂或家庭场景直接使用
  3. 打开即用，不依赖复杂环境

## 2) 目标受众（默认）

- 如无特别说明，默认受众为**小学阶段孩子**。
- UI/交互默认应符合低龄用户特征：
  - 信息层级清晰，操作路径短
  - 按钮大、可点击区域大、触摸友好
  - 文字简洁直白，避免专业术语
  - 色彩活泼但不过度刺激，保持可读性
  - 关键反馈明显（如成功/失败、下一步提示）

## 3) 使用场景（默认）

- 如无特别说明，默认场景是：
  - 学生在**家长手机或平板**上使用
  - 链接可能从**钉钉群**直接打开
- 因此默认要求：
  - **移动端优先（Mobile First）**
  - 兼容小屏幕（常见手机宽度）
  - 首屏核心功能可直接操作
  - 避免依赖键盘复杂输入，尽量提供触控友好输入方式

## 4) 部署方式（强约束）

- 项目需要可直接部署到 **Cloudflare Pages**。
- 当前仓库采用**零构建**静态站点模式，需保持：
  - Framework preset: `None`
  - Build command: 留空
  - Build output directory: `/`
- 开发时请遵守：
  - 优先使用原生 HTML/CSS/JS
  - **尽可能不要引入前端构建框架**（如需要打包构建的工具链）
  - 不要引入必须 Node 构建后才能运行的方案

## 5) 现有架构约定（依据 README 与当前代码）

- 仓库是“多个独立静态小工具”的集合：
  - 根目录 `index.html`：工具导航入口
  - 每个工具在独立子目录（例如 `lucky/`、`clock/`）
- 每个子工具应尽量自包含：
  - `index.html` + 必要的 `css/js` 资源
  - 可独立访问路径：`/<tool-name>/`
- 新增工具时，默认同步更新根目录导航页 `index.html`。

## 6) 代码实现偏好

- 优先简单、直接、可维护的实现，不做过度抽象。
- 与现有代码风格保持一致（纯静态、少依赖、可离线优先）。
- 对儿童使用场景重要的能力可优先考虑：
  - 本地存储（如 `localStorage`）
  - 明确的操作提示
  - 可恢复/可重置的数据操作
- 如涉及 PWA/离线能力，沿用当前静态资源缓存思路，避免复杂化。

## 7) 交付检查清单（Code Agent 执行前后自检）

在未被特别指示时，默认满足以下检查：

1. 是否面向小学生可理解、可操作？
2. 是否在手机小屏上可正常使用？
3. 是否不依赖构建流程、可直接静态部署到 Cloudflare Pages？
4. 是否遵循“一个工具一个子目录”的现有架构？
5. 若新增工具，是否更新了根目录导航入口？
